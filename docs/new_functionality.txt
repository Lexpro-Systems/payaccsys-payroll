SPRINT 2020-04-29
=================

- Change Password <DONE>

  Allow users to change their password from the 'user menu'.  When the option is selected show a screen where the user can enter
  a new password.  After the user has changed their password an email must be sent to notify them of the action.

- Session Refresh Tokens <DONE>

  Update the session system to make use of refresh tokens.

- Invalidate Session on Password Change <DONE>

  When a user changes his password either by useing the 'Reset Password' or 'Change Password' options, all sessions must be
  invalidated to ensure they can no longer be used.

- Reset Password Messaging <DONE>

  Improve the information displayed to the user when resetting their password to better inform them of the process and its results.

  Example Mesage: Verification Panel

      We sent a verification code to <user email address>.  Please enter it to continue.

- Store Company Access Details <DONE>

  Update the system to no longer delete rows from the user_company_access table so that it is possible to see who had access to
  a company and when.  This should be done by marking the row as 'revoked'.  It is important to store the date that the access was
  revoked so that it is possible to tell from which date to which date the user had access to the company.  Update login and session
  refresh functions to take this revoked flag into account.

- Caps Lock Notification <DONE>

  Improve the password component to show a message when the caps lock is on.

- Hide Dismissed Employees <DONE>

  Hide dismissed employees on the list employees panel.  The user must have a filter that allow them to show all employees,
  dismissed employees only, or employed employees only.  Defualt should be employed employees only.

- Per KM On Payslip Items <DONE>

  When showing payslip items on the 'Income / Deductions' panel show 'per km' if the item is of a type that uses that unit.

- Use Alias To Refer to Employees <DONE>

  Use alias instead of first and last name where appropriate
